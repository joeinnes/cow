swt clones your entire repo — node_modules, build outputs, caches, .env files and all — in milliseconds using APFS copy-on-write. Every agent gets a full, ready-to-run workspace with zero disk overhead.

The problem

Every other option makes
you wait, or gives up half the repo

Parallel agents need isolated workspaces. The existing tools range from inconvenient to unworkable.

git worktree
$ git worktree add ../feat feat/login
Preparing worktree (new branch 'feat/login')
HEAD is now at a3f8d12
⚠ node_modules/ — not included
⚠ dist/, .next/ — not included
⚠ .env — not included
# npm install before agent can start

Only tracked files are checked out. Every workspace needs a fresh install — minutes before any agent can do anything.

full clone (cp -r)
$ cp -r . ../feat-login
copying node_modules/ ...
copying .git/ ............
copying dist/ .........
2 min 18 sec
+4.8 GB disk used
# × 10 agents = 48 GB, 23 mins

Includes everything, but block-copies every byte. Ten agents on a 5 GB repo costs 50 GB and the better part of half an hour.

containers
$ docker build -t agent-env .
Step 1/12 : FROM node:20
Step 2/12 : WORKDIR /app
Step 3/12 : COPY package*.json ./
Step 4/12 : RUN npm ci
Step 5/12 : ...
# overkill for a directory copy

Conceptually clean but operationally heavy. Networking, volumes, and image rebuilds for something that should be a single command.

How swt works

Copy-on-write
all the way down

  • APFS clonefile(2) creates a full copy of every file in constant time — the kernel shares blocks until something writes.
  • A 5 GB repo cloned ten times still costs ~5 GB until agents modify files. Only changed blocks consume new space.
  • node_modules, dist, .next, .env, caches — all immediately present. No install step, no build step.
  • Each workspace is an independent git repo. Checkout, commit, push — no shared-state worktree bookkeeping.
  • Post-clone cleanup removes stale pid files, sockets, and lock files. Project-specific patterns via .swt.json.

CLI reference

Three commands,
most workflows covered

Auto-named workspaces, branch creation, dirty-state warnings. Simple by default, configurable when needed.

~/projects/myapp $ swt create
Cloning /Users/joe/projects/myapp ...
Created workspace 'agent-1' at /Users/joe/.swt/workspaces/agent-1
 
~/projects/myapp $ swt create agent-2 --branch feat/search
Cloning /Users/joe/projects/myapp ...
Created workspace 'agent-2' at /Users/joe/.swt/workspaces/agent-2
 
# name is optional — auto-increments as agent-1, agent-2, ...
# --branch creates the branch if it doesn't exist
# --source <path> to clone from a different repo
# --no-clean to skip pid/sock cleanup
~/projects/myapp $ swt list

MCP integration

Agents that
manage themselves

swt mcp runs a stdio MCP server. Add it to your ~/.claude/settings.json and Claude Code can create, list, and remove workspaces as tool calls — no shell access required, no human in the loop.

  • create_workspace Clone a repo, return the workspace path
  • list_workspaces Return workspace list as structured JSON
  • remove_workspace Remove by name with dirty-state info
  • workspace_status Branch, VCS status, disk delta
Coming soon — tracked in backlog
~/.claude/settings.json
settings.json
{
  "mcpServers": {
    "swt": {
      "command": "swt",
      "args": ["mcp"]
    }
  }
}

Once registered, Claude Code discovers the tools automatically on startup. No slash commands, no custom prompts — just structured tool calls.

Install

Up and running in
thirty seconds

Requires macOS 12 Monterey or later with an APFS volume — the default on every modern Mac.

Homebrew recommended 
$ brew tap joeinn.es/tap
$ brew install sparse-worktree
# verify
$ swt --version
From source requires Rust 
$ git clone https://github.com/
    joeinn.es/sparse-worktree
$ cd sparse-worktree && make install
